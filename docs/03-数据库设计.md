# 03 数据库设计

## 1. 关系型数据库说明

本项目使用 Supabase PostgreSQL。PostgreSQL 与 MySQL、SQLite 同属关系型数据库：数据以表组织，使用主键唯一标识记录，使用外键表达表关系，并通过 SQL 完成查询和 CRUD。

选择 PostgreSQL/Supabase 的原因：

- 需要公网多用户持久化，SQLite 的单文件模式不适合作为本项目生产数据库。
- 需要 RLS、触发器、函数、枚举和约束，PostgreSQL 支持完整。
- Supabase 将 PostgreSQL、Auth 和会话集成，课程项目无需另建认证服务。
- 相比自行部署 MySQL，减少服务器、连接池和备份配置，但数据模型仍遵循相同的关系型设计原则。

## 3. 六张业务表

### profiles

| 字段 | 类型 | 约束/说明 |
| --- | --- | --- |
| `id` | `uuid` | 主键，同时外键关联 `auth.users.id` |
| `email` | `text` | 非空，建立小写唯一索引 |
| `full_name` | `text` | 可空 |
| `role` | `user_role` | `buyer/operator/admin`，默认 buyer |
| `created_at/updated_at` | `timestamptz` | 创建与更新时间 |

### rfqs

除主键、唯一 RFQ 编号和 `buyer_id` 外键外，包含产品名称、分类、参考 URL、描述、文件名、材质、尺寸、颜色、Logo、包装、附加要求、数量、目标价、目的国、Marketplace、交付日期、样品、履约方式、状态和已选报价。数量/价格必须大于 0；状态是八值枚举。

### suppliers

包含匿名 `supplier_code`、内部公司名、联系邮箱、地区、能力、备注、启用状态和时间戳。真实公司名只允许 Staff 数据路径读取和维护，Buyer 只看到报价中的匿名 Label。

### supplier_quotes

关联 RFQ、Supplier 和创建人；保存单价、MOQ、样品费、交期、包装和备注。同一 RFQ 下同一供应商只能有一份报价。`rfqs.selected_quote_id` 反向关联已选择报价，并由触发器检查报价必须属于当前 RFQ。

### timeline_events

关联 RFQ 和创建人，保存可选状态、标题、详情、事件时间与排序值。RFQ 创建后触发器自动写入 Requirements Submitted 节点。

### attachments

保存 RFQ、上传人、文件名、存储路径、MIME、大小和分类。Course v1.0 仅使用附件元数据，不实现真实 Storage 上传。

## 4. CRUD 与完整性

| 对象 | Create | Read | Update | Delete |
| --- | --- | --- | --- | --- |
| Profile | Auth 触发器 | 自己/Staff | Admin | 级联 Auth 用户 |
| RFQ | Buyer/Staff | Owner/Staff | Owner 授权字段/Staff | 未开放前端删除 |
| Supplier | Staff | Staff | Staff | Staff |
| Quote | Staff | 相关 Buyer/Staff | Staff；Buyer 仅选中 | Staff |
| Timeline | 自动/Staff | 相关 Buyer/Staff | Staff | Staff |
| Attachment | Owner/Staff | 相关 Buyer/Staff | Owner/Staff | Owner/Staff |

## 5. RLS 策略

- 六表全部启用 RLS，并撤销 `anon` 业务表权限。
- `private.is_staff()` 和 `private.is_admin()` 在数据库内读取当前角色。
- Buyer 查询通过 `buyer_id = auth.uid()` 或关联 RFQ 判断。
- Supplier 写入、Quote 写入、Timeline 写入只允许 Operator/Admin。
- Profile 更新只允许 Admin；用户不能修改自己的角色。
- `set_user_role` 是 Security Definer RPC，但函数内部再次要求 Admin。

## 6. 触发器与索引

- 六个 `set_updated_at` 触发器自动更新时间。
- `on_auth_user_created` 为新 Auth 用户创建 Buyer Profile。
- `on_rfq_created` 创建首条时间线。
- `rfqs_protect_staff_fields` 防止 Buyer 修改 owner、编号和状态，并校验已选报价归属。
- 索引覆盖 buyer、状态、创建时间、RFQ 报价、供应商报价、时间线和附件外键。
