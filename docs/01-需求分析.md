# 01 需求分析

## 1. 项目概述

SourceBridge 是跨境采购工作流课程原型。它连接海外买家与中国采购运营人员，用结构化 RFQ 管理需求、报价、成本和履约过程。Course Release v1.1 的目标是展示现代前端框架、类型系统、公开 Buyer 注册、认证、关系型数据库和响应式设计的综合应用，而不是开展商业获客。

## 2. 用户角色

| 角色 | 核心目标 | 主要权限 |
| --- | --- | --- |
| 匿名访客 | 了解项目和课程能力 | 浏览首页、How it works、Capabilities、公开只读 Demo 和登录页 |
| Buyer | 提交并跟踪自己的采购需求 | 创建 RFQ；只读/修改自己的 RFQ；查看自己的报价、附件和时间线；选择报价 |
| Operator | 执行采购运营工作 | 查看全部 RFQ；维护供应商；添加报价和时间线；更新 RFQ 状态；不能修改角色 |
| Admin | 维护系统与权限 | 拥有 Operator 能力；查看 Profile；修改已注册用户角色 |

新注册用户在数据库触发器中固定为 `buyer`，不能通过前端参数或 Auth metadata 选择管理员。Course Release v1.1 开放课程演示注册，但暂时不验证邮箱所有权；Operator 和 Admin 仍只能由现有 Admin 授权。

## 3. 使用场景

1. 匿名访客从首页了解四步采购流程，并打开 `/demo` 查看完整演示 RFQ。
2. Buyer 使用课程账号登录，在 Dashboard 查看自己的 RFQ。
3. Buyer 用四步骤表单提交产品、规格和商业要求，刷新后数据仍存在。
4. Operator/Admin 查看新 RFQ，录入匿名供应商报价并更新履约节点。
5. Buyer 比较报价、选择偏好方案并查看成本变化。
6. Admin 查看用户资料并在必要时调整角色。

## 4. 功能需求

### 4.1 公开展示

- 首页包含 Hero、流程、能力、价值和 Prototype Service Targets。
- `/demo` 提供稳定的只读课程案例，两份匿名报价、已选择报价、成本估算和完整时间线。
- 所有模拟内容明确写明 Demo/Illustrative，不冒充真实经营数据。

### 4.2 认证与会话

- 邮箱密码登录、退出和会话保持。
- 未登录访问业务路由时跳转 `/login`。
- 未配置 Supabase 时 API 安全关闭并返回 503。
- 配置 Supabase 但未登录时 API 返回 401。

### 4.3 Buyer 工作流

- Dashboard 展示统计和当前用户 RFQ。
- 多步骤 RFQ 表单包含 Product Basics、Specifications、Commercial Requirements、Review。
- 必填、格式、数值、日期和产品分类进行前端及服务端校验。
- Buyer 只能访问自己的 RFQ 及其报价、时间线和附件。

### 4.4 运营后台

- RFQ 数量、状态统计和全部 RFQ 列表。
- 创建/编辑供应商、添加报价、添加时间线、更新状态。
- Operator 不能修改角色；Admin 可以通过受保护 RPC 修改角色。

## 5. 非功能需求

- **安全性**：服务端鉴权 + PostgreSQL RLS 双重保护；匿名用户无业务表读取权限。
- **可靠性**：数据写入 Supabase；刷新、退出和重新登录后仍存在。
- **可维护性**：19 个组件、独立类型、Mock 数据、验证器和数据库类型。
- **可访问性**：表单标签、错误提示、语义化结构、焦点样式和键盘操作。
- **响应式**：390px 与 1440px 无横向溢出，重要内容不被 Sticky Header 遮挡。
- **可测试性**：18 项自动化测试、真实公开注册与多账号 RLS 集成脚本和安全扫描。

## 6. 项目价值

对业务场景而言，SourceBridge 展示了如何把分散沟通转化为可追踪流程。对课程学习而言，它覆盖组件化、状态管理、表单、路由、API、认证、数据库建模、权限、安全、测试和部署，是一个完整的前端框架工程实践案例。
