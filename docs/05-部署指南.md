# 05 部署指南

## 1. 环境变量

本地 `.env.local` 和 Sites 运行环境只配置以下名称，文档和仓库不保存真实值：

```dotenv
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
```

不得在浏览器、`NEXT_PUBLIC_` 变量、Git、README 或 Sites 源码中放入：

- Supabase service role / secret key
- PostgreSQL 数据库密码或连接串
- 测试账号密码
- Sites 临时仓库凭据或绕过令牌

## 2. Supabase 配置

1. 创建 Supabase Free 项目。
2. 在 SQL Editor 依次执行 `supabase/migrations/202607110001_sourcebridge_phase2.sql` 和 `supabase/migrations/202607110002_course_v1_1_registration.sql`。
3. 确认六张业务表、索引、触发器、函数、RLS Policy 已建立。
4. Course v1.1 保持 Email + Password 和 Allow new users to sign up 开启，保持匿名登录关闭。
5. 因尚未配置自定义 SMTP，Course Demo 暂时关闭 Confirm email；注册后立即创建会话。商业化前必须重新开启邮箱确认。
6. 使用 `supabase/admin/promote_user_by_email.sql` 或 Admin RPC 提升已注册账号，不在源码硬编码邮箱。

## 3. Auth URL

生产 Site URL：

```text
https://sourcebridge-sourcing-demo.buxiu790.chatgpt.site
```

允许的 Redirect URL 至少包含：

```text
http://localhost:3000/auth/callback
http://localhost:3000/auth/confirm
https://sourcebridge-sourcing-demo.buxiu790.chatgpt.site/auth/callback
https://sourcebridge-sourcing-demo.buxiu790.chatgpt.site/auth/confirm
```

Course v1.1 的 `/forgot-password` 不提交重置请求。商业化前配置自定义 SMTP，重新开启 Confirm email，并验证确认邮件、找回密码链接和发件域名后，才能恢复密码找回流程。

## 4. 本地构建与检查

```bash
npm install
npm run lint
npx tsc --noEmit
npm test
npm audit --omit=dev
```

`npm test` 已包含 `vinext build`。真实环境验收使用：

```bash
npm run test:live
npm run test:signup-live
```

该脚本只读取公开 Supabase 参数和本机 `.private/sourcebridge-live-sessions.json` 普通用户会话，临时 Supplier/Quote/Timeline 数据在 `finally` 中清理。

## 5. Sites 发布步骤

1. 检查 `.openai/hosting.json` 并复用已存在的 `project_id`，不得重复创建站点。
2. 确认 Git 工作区只有课程版本修改，`.env.local` 和 `.private/` 未跟踪。
3. v1.0 标签保持不动；Course v1.1 使用提交 `release: SourceBridge Course v1.1 registration` 并创建新标签 `course-v1.1`。
4. 将该 Commit 推送到 Sites 绑定的源码仓库。
5. 基于同一 Commit 构建并保存 Sites Version。
6. 设置两项生产环境变量；Secret 值只存 Sites 配置。
7. 部署已保存版本，确认部署成功后把访问模式切为 Public。
8. 用公开 URL 做匿名、认证、角色、响应式和控制台冒烟测试。

## 6. 常见错误

| 问题 | 排查方法 |
| --- | --- |
| 登录后仍回到 Login | 检查两项公开环境变量、Cookie、Supabase Site URL/Redirect URL |
| API 返回 503 | 当前运行环境未配置 Supabase URL/Anon Key |
| API 返回 401 | 会话缺失或过期，重新登录 |
| Admin 页面回 Dashboard | Profile 角色不是 operator/admin，或服务端角色检查失败 |
| Buyer 看不到 RFQ | 检查 `buyer_id` 是否等于 Auth UID 和 RLS Policy |
| 报价选择返回 403 | RFQ 不属于当前 Buyer，或 Quote 不属于该 RFQ |
| 生产页面客户端资源 404 | 确保使用最新完整构建产物；重新构建并保存同 Commit Sites Version |
| 注册后没有会话 | 检查 Course Demo 的 Confirm email 是否暂时关闭；商业化版本不得依赖此配置 |
| 无法找回密码 | Course v1.1 在自定义 SMTP 可用前主动禁用密码恢复 |
| 创建第 6 条 RFQ 被拒绝 | Course Demo 每个 Buyer 最多 5 条 RFQ，属于服务端和数据库强制限制 |
| Sites 仍要求 ChatGPT 登录 | 检查 Sites Access Mode 是否已切换为 Public |
